# Portfolio Methodology

## Why this portfolio exists
This portfolio demonstrates how domain-specific AI projects can be framed, designed, implemented, evaluated, and communicated.

## Project Documentation Standard
1. Problem Framing
2. Task Source
3. Domain Constraints
4. Technical Route
5. System Architecture
6. Data Flow
7. Evaluation Plan
8. Results and Limitations
9. Role-based Interpretation

## What I intentionally do not show
- Core production code
- Proprietary algorithms
- Real customer data
- Internal parameters
