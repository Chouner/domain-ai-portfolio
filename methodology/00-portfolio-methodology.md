# Portfolio Methodology / 作品集方法论

## Why this portfolio exists / 为什么需要这个作品集

This portfolio demonstrates how domain-specific AI projects can be framed, designed, implemented, evaluated, and communicated.

这个作品集用于展示专业领域 AI 项目如何被定义、设计、实现、评估和表达。

## Project Documentation Standard / 项目文档标准

Each project is documented using the following structure:

每个项目按照以下结构记录：

1. Problem Framing / 问题定义
2. Task Source / 任务来源
3. Domain Constraints / 领域约束
4. Technical Route / 技术路线
5. System Architecture / 系统架构
6. Data Flow / 数据流
7. Evaluation Plan / 评估方案
8. Results and Limitations / 结果与限制
9. Role-based Interpretation / 岗位化表达

## What I intentionally do not show / 有意不展示的内容

- Core production code / 核心生产代码
- Proprietary algorithms / 公司专有算法
- Real customer data / 真实客户数据
- Internal parameters / 内部参数
