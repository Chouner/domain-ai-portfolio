# Domain AI Project Lifecycle / 专业领域 AI 项目生命周期

## 1. Task Source / 任务来源

- Customer requirement / 客户需求
- Internal expert pain point / 内部专家痛点
- Data-processing bottleneck / 数据处理瓶颈
- Product opportunity / 产品机会
- Technical reserve / 技术储备
- Research exploration / 研究探索

## 2. Problem Framing / 问题定义

- Who has the problem? / 谁遇到了问题？
- What is the current workflow? / 当前流程是什么？
- Where is the bottleneck? / 瓶颈在哪里？
- Why is AI needed? / 为什么需要 AI？
- What cannot be solved by simple automation? / 哪些问题不是简单自动化能解决的？

## 3. Domain Constraints / 领域约束

- Physical constraints / 物理约束
- Data quality constraints / 数据质量约束
- Sensor constraints / 传感器约束
- Operational constraints / 作业约束
- Expert knowledge constraints / 专家知识约束

## 4. Solution Design / 方案设计

- Rule-based method / 规则方法
- Algorithm module / 算法模块
- RAG / RAG
- Agent workflow / Agent 工作流
- Multimodal model / 多模态模型
- Hybrid AI + traditional algorithm / AI 与传统算法混合方案

## 5. MVP Validation / MVP 验证

- Minimal dataset / 最小数据集
- Minimal workflow / 最小工作流
- Minimal interface / 最小界面
- Minimal evaluation metric / 最小评估指标

## 6. Evaluation / 评估

- Accuracy / 准确率
- Recall / 召回率
- Stability / 稳定性
- Human intervention rate / 人工介入率
- Traceability / 可追溯性
- Latency / 延迟
- Cost / 成本

## 7. Delivery / 交付

- Documentation / 文档
- Demo / Demo
- API or interface / API 或界面
- Test report / 测试报告
- Handover checklist / 交接清单

## 8. Iteration / 迭代

- Failure cases / 失败案例
- Missing data / 缺失数据
- Model limitation / 模型限制
- Next version / 下一版本
