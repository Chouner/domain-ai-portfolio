# 技术评分模型

每个采集到的技术在进入实验 backlog 前都需要评分。

| 维度 | 分值 | 衡量方式 |
|---|---:|---|
| 场景匹配度 | 1-5 | 是否匹配当前声学/AI 处理问题 |
| 证据强度 | 1-5 | 论文质量、代码可用性、benchmark 质量、可复现性 |
| 技术成熟度 | 1-5 | 研究想法、原型、可复用库、生产可用 |
| 集成成本 | 1-5 | 数据、算力、依赖、接口和维护成本；成本越低分越高 |
| 验证清晰度 | 1-5 | 是否能用现有数据和指标验证成功 |
| 风险 | 1-5 | license、专利、幻觉、安全和可靠性风险；风险越低分越高 |
| 复用性 | 1-5 | 是否能跨模块或跨项目复用 |

## 决策规则

| 加权分 | 决策 |
|---:|---|
| >= 4.2 | 立即实验 |
| 3.5-4.2 | 储备并跟踪 |
| 2.8-3.5 | 人工复审 |
| < 2.8 | 拒绝或归档 |
