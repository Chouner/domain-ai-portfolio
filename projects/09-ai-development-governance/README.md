# AI Development Governance / AI 协同开发规范

## One-line Summary / 一句话概述

A governance framework for using AI tools in long-running software development tasks.

一套用于长期软件开发任务中使用 AI 工具的协同开发规范。

## Core Problem / 核心问题

In long development tasks, AI tools may misunderstand requirements, lose context, change architecture without permission, or produce inconsistent code.

在长周期开发任务中，AI 工具可能误解需求、丢失上下文、未经确认修改架构，或生成风格不一致的代码。

## Core Innovation / 核心创新

Transform AI coding from ad-hoc prompting into a controlled engineering process with context management, requirement freeze, task boundary, change confirmation, code review, and acceptance testing.

将 AI 编程从随问随答转化为有上下文管理、需求冻结、任务边界、变更确认、代码评审和验收测试的工程流程。
